**Step 1 — Add imports** at the top, after existing imports:

```typescript
import compression from 'compression';
import timeout from 'connect-timeout';
```

Add a `haltOnTimedout` helper function before the `app` is created (or directly above first `app.use` call):

```typescript
function haltOnTimedout(req: Request, res: Response, next: NextFunction): void {
  if (!req.timedout) next();
}
```

**Step 2 — Read timeout from env:**

```typescript
const REQUEST_TIMEOUT_MS = Number(process.env.REQUEST_TIMEOUT_MS) || 30_000;
```

**Step 3 — New middleware order in src/app.ts:**

The complete order must be:

```
app.use(helmet());
app.use(cors(corsOptions));
app.use(timeout(REQUEST_TIMEOUT_MS));   // <-- NEW: start timeout clock
app.use(compression());                  // <-- NEW: compress all responses
// ... rate limiter (existing) ...
app.use(morgan('combined'));
app.use(express.json({ limit: '300mb' }));
app.use(express.urlencoded({ extended: true, limit: '300mb' }));
app.use(haltOnTimedout);                 // <-- NEW: stop timed-out requests before routes

// ... health check, swagger, root, all route registrations (unchanged) ...

// ... notFound ...
app.use(haltOnTimedout);                 // <-- NEW: second guard before errorHandler
app.use(errorHandler);
```

**Step 4 — Protect the large-upload push route from premature timeout:**

The `/api/offline-sales/push` route accepts up to 300MB bodies which can legitimately take longer than 30s. Add a route-specific override BEFORE `mountOfflineSales` is called:

```typescript
// Disable global timeout for the large-body push endpoint
app.use('/api/offline-sales/push', (req: Request, _res: Response, next: NextFunction) => {
  req.setTimeout(0); // disable socket timeout for this route
  next();
});
```

Place this line immediately before `mountOfflineSales(app, '/api/offline-sales')`.

**Swagger UI pitfall:** After adding compression, start the dev server and load `/api-docs` in a browser. If the Swagger UI shows blank or throws a JS parse error, add a compression filter to skip the `/api-docs` path:

```typescript
app.use(compression({
  filter: (req, res) => {
    if (req.path.startsWith('/api-docs')) return false;
    return compression.filter(req, res);
  },
}));
```

Apply the filter preemptively — it costs nothing and prevents a potential production incident.

**Step 5 — Update env.example:** Append after the existing logging section:

```
# Request timeout — all endpoints return 503 if they exceed this threshold (milliseconds)
# Exception: /api/offline-sales/push is exempt (accepts large upload bodies)
REQUEST_TIMEOUT_MS=30000
```
